## Leon Wang Photo 网站未来架构升级规划

## 项目简介

当前网站定位：

* 个人摄影作品展示
* 生活记录（Photo & Life）
* 个人博客
* 技术分享

当前技术栈：

* Next.js
* PostgreSQL
* Docker
* Caddy
* 腾讯云服务器
* 域名：[www.leonwang.cc](http://www.leonwang.cc)

---

# 第一阶段（当前）

## 架构

```text
Internet
    ↓
www.leonwang.cc
    ↓
Caddy
    ↓
Next.js
    ↓
PostgreSQL
```

## 图片存储

```text
/uploads
    ├── photo1.jpg
    ├── photo2.jpg
    └── photo3.jpg
```

图片存储于服务器本地磁盘。

## 优点

* 架构简单
* 成本低
* 维护方便

## 缺点

* 图片占用服务器磁盘
* 扩容不方便
* 服务器迁移需要同步图片

---

# 第二阶段（家庭服务器）

目标：

降低云服务器成本，实现家庭私有部署。

## 推荐硬件

### 方案一：Mac mini（推荐）

推荐型号：

* Mac mini M4
* Mac mini M4 Pro

优点：

* 功耗低
* 性能强
* 稳定性高
* Docker 支持良好

### 方案二：Mini PC

推荐：

* Intel N100
* Intel N305

优点：

* 价格便宜
* 功耗低

---

## 架构

```text
Cloudflare
      ↓
Cloudflare Tunnel
      ↓
Mac mini / Mini PC
      ↓
Docker
      ↓
Next.js
      ↓
PostgreSQL
```

## 优点

* 不需要公网 IP
* 不需要端口映射
* 自动 HTTPS
* 无服务器费用

---

# 第三阶段（Cloudflare CDN）

目标：

降低家庭宽带压力。

## 架构

```text
用户
 ↓
Cloudflare CDN
 ↓
Cloudflare Tunnel
 ↓
家庭服务器
```

## 图片缓存策略

服务器返回：

```http
Cache-Control: public, max-age=31536000, immutable
```

Cloudflare 自动缓存：

* jpg
* jpeg
* png
* webp
* avif

## 优点

* 图片访问速度提升
* 降低家庭带宽消耗
* 提高全球访问速度

---

# 第四阶段（Cloudflare R2）

目标：

将图片与网站程序解耦。

## 架构

```text
                    ┌────────────┐
                    │ Cloudflare │
                    │    CDN     │
                    └─────┬──────┘
                          │
          ┌───────────────┴───────────────┐
          │                               │
          ▼                               ▼

    www.leonwang.cc                 img.leonwang.cc
          │                               │
          ▼                               ▼

      Next.js                       Cloudflare R2

          │
          ▼

      PostgreSQL
```

---

## 网站职责

### Next.js

负责：

* 首页
* 相册管理
* 文章系统
* 用户系统
* 上传管理

### PostgreSQL

负责：

* 相册数据
* 文章数据
* 标签分类

### R2

负责：

* 原图
* 缩略图
* 封面图
* WebP 文件

---

## 图片处理流程

```text

     ↓
```

```text
2048px
WebP
```

```text
RAW/JPG
```
